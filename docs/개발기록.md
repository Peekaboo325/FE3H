# 개발기록 (DEVLOG) — FE3H v2.0

> **이 문서의 목적**: Claude Code 컨텍스트를 비워도 개발을 이어가기 위한 '살아있는 기록'.
> 새 세션은 CLAUDE.md → 이 문서를 읽고 현재 상태를 복원한다. **의미 있는 변경 후엔 이 문서를 갱신한다.**
> 최종 갱신: 2026-06-05

---

## 1. 한눈에 — 지금 무엇이 돌아가나

루트에 v2.0 인터랙티브 픽션 앱이 서 있고, Vercel에 배포되어 실제로 동작한다.

핵심 고리:
```
[화면(React)] → [서버 한 겹] → [Claude Opus 4.8] → 본문 스트리밍 → [Supabase 저장] → 화면 복원
                   └ 주입: 박제 세계관(캐싱) + 활성 견문록 + 활성 인물
```

- **서버 한 겹**: 로컬은 Express(`server/index.mjs`), 배포는 Vercel 서버리스 함수(`api/*.mjs`). 둘이 같은 일.
- **원본 저장소 = Supabase**(클라우드). 브라우저는 Supabase에 직접 접근하지 않고 항상 서버(/api)를 거친다.
- **모델 분배**: 메인 본문 = Claude Opus 4.8(effort 'low', adaptive thinking). 나머지(편지·요약·추출 등) = Gemini(키만 설정, **아직 미사용**).

## 2. 기술 스택 / 운영

- Vite + React 19 + TypeScript. (Zustand 미도입 — 아직 상태가 단순)
- 로컬 개발: `npm run dev` (vite 5173 + express 8787, vite가 `/api`를 8787로 프록시).
- 빌드: `npm run build` (= `npm run bake && vite build`).
- **세계관 굽기**: `npm run bake` → `scripts/bake-worldview.mjs`가 `md/` 박제 4종을 `lib/worldview.mjs`로 구워 넣음(dev·build 시 자동). **md 고치면 재배포해야 반영.**
- **배포 = `main`에 push** → Vercel 자동 배포. 개발 브랜치 `claude/wizardly-dirac-WjCg3`에서 작업 → `main`에 fast-forward 머지 → push(양쪽).
- 배포 전 항상 `npm run lint`(tsc) + `npm run build` 통과 확인.

### 환경 변수 (Vercel 설정 + 로컬 `.env`)
- `ANTHROPIC_API_KEY` — 클로드 본문. **(필수)**
- `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` — 저장소. **service_role(비밀) 사용. anon 아님.** (없으면 저장 없이 동작)
- `GEMINI_API_KEY` — 설정만 됨, 아직 쓰는 코드 없음.
- ⚠️ Vite는 `VITE_` 접두사 변수를 브라우저에 노출 → 비밀 키에 절대 `VITE_` 붙이지 말 것.

### Supabase 표 (SQL Editor에서 1회 실행, RLS on·정책 없음 → service_role만 접근)
- `turns(id, role, content, created_at)` — 이야기.
- `characters(id, name, english_name, aliases, faction, title, appearance, personality, combat, notes, life_status, is_active, thumbnail, sort_order, created_at)`.
- `lore(id, title, category, body, is_active, sort_order, created_at)` — 견문록(연재 고유 설정).

### 진단
- `/api/turns`, `/api/characters`, `/api/lore`를 브라우저로 열면 `{dbReady, error, ...}`가 보임.
  - `dbReady:false` → 키 미설정/이름오타/미배포. `error`에 "relation ... does not exist" → 표 없음. "row-level security/permission" → anon 키 잘못 사용.

## 3. 데이터 분류 (박제 / 커스텀) — 핵심 설계

| 층 | 내용 | 처리 |
|---|---|---|
| **박제** (원작 정사·작법) | 세계관 규약 · 프로젝트 지침 · 성인향 묘사 지침 · 고유명사 사전 | `md/` → bake → 시스템 프롬프트로 **항상 주입 + 프롬프트 캐싱** |
| **커스텀 — 인물** | 인물 설정 사전 · 주요 인물 노트 | **인물 기능**(characters 표 + UI) |
| **커스텀 — 세계** | 연재 고유 설정 | **견문록 기능**(lore 표 + UI) |

- 새 고유명사는 고유명사 사전(박제)을 건드리지 않고 커스텀(인물/견문록)에 추가한다.

## 4. 빌드된 기능 + 파일 지도

- **이야기 생성·저장·복원**: `api/story.mjs`·`server`(스트리밍, turns 저장), `src/App.tsx`(켤 때 `/api/turns`로 복원).
- **세계관 주입**: `lib/worldview.mjs`(박제, 캐시 블록) + `lib/charContext.mjs`(활성 인물) + `lib/loreContext.mjs`(활성 견문록). 인물·견문록 블록은 **박제 뒤에 붙여 캐싱 유지**.
  - 인물 주입 정책 = **B(핵심+참조라벨)**: 이름·이명·소속·정의·성격은 늘, 외양·무기·비고는 '참조용 — 필요할 때만' 라벨.
  - 견문록 주입: '원작 기본값보다 우선', '설명조 나열 금지'.
- **인물 프로필**: `src/Characters.tsx` + `src/imageUtils.ts`(업로드 이미지를 600px·WebP로 변환, 인라인 저장) + `lib/db.mjs` CRUD. 헤더 '인물' 버튼.
- **견문록**: `src/Lore.tsx` + `lib/db.mjs` CRUD. 헤더 '견문록' 버튼.
- **목록 캐시**: `src/useCachedList.ts`(stale-while-revalidate: 메모리 + IndexedDB) → `useCharacters`·`useLore`가 얇은 래퍼. `src/idbCache.ts`(의존성 없는 IndexedDB KV). **원본은 Supabase, 캐시는 빠른 사본.**
- **폰트**: 국문 SUIT(jsDelivr CDN @font-face), 영문 Hanken Grotesk(Google Fonts), 서명·명문 Pinyon Script. 본문 속 라틴(=포드라 문자)은 `App.tsx`의 `포드라문자_렌더`가 자동으로 `.script`(Pinyon)로 렌더.

## 5. 주요 결정과 이유 (왜 이렇게 했나)

- **md(문서) vs .ts(코드)**: 세계관을 .md로 두는 건 클로드 입장에선 동일(텍스트만 봄). 단지 비개발자가 편집하기 쉽고, 콘텐츠(유저 영역)와 코드(시스템) 분리. 코드가 직접 import 못 하니 bake로 다리를 놓음.
- **Supabase = 창고(DB)**, Vercel = 앱·서버. Supabase를 '우리 서버'로 쓰지 않고 서버 뒤 창고로 둠(키 숨김).
- **브라우저 캐시 = IndexedDB**(localStorage 5MB·동기 한계 회피). 단 원본은 Supabase, 캐시는 임시 사본.
- **썸네일 = WebP**(wbmp 아님 — wbmp는 흑백 1비트 구식). 600px 리사이즈. DB 인라인 저장(소량이라 Storage 버킷 생략).
- **인물 주입은 '전부'가 아니라 핵심+참조라벨**: 다 넣으면 모델이 외양·무기를 습관적으로 읊어 글이 산만 → 프로젝트 지침의 "아는 걸 다 꺼내지 말 것"과 충돌.

## 6. 알려진 한계 · 다음 할 일

- ⚠️ **컨텍스트 윈도우 미구현**: 현재 프런트가 **전체 대화(turns)를 매 요청 전송**, 서버도 클리핑 안 함. 회차가 쌓이면 비용·토큰 폭증. CLAUDE.md 원칙대로 **'누적 줄거리 요약 + 최근 1~2화'** 로 전환 필요(요약은 Gemini).
- **선택 주입(비용)**: 고유명사 사전(~17KB)이 매 회차 통째로 주입됨. 등장 이름만 골라 넣기 / 견문록·인물도 '이 장면에 필요한 것'만. 후보: 키워드 매칭 → pgvector(임베딩) 또는 Gemini 경량 셀렉터. (지금 규모면 과잉일 수 있음 — 항목 많아질 때)
- **Gemini 미연결**: 편지·줄거리 요약·인물 추출·해시태그 등 기계적 처리용. 키만 있음.
- **편지/서신 기능**: legacy 7종. Pinyon Script `.script` 서명체가 여기서 진가 발휘.
- **연대기·기억 시스템**: legacy 참고.
- **비용 상한**: Anthropic Console 워크스페이스 spend limit 미설정.
- **본문 문체**: 박제 프롬프트가 Gemini Flash 기준이라 클로드용 톤 재조정 여지. 실제 출력 보고 조정.

## 7. 작업 컨벤션 (이 프로젝트에서)

- 항상 **한국어**로, **비개발자**가 따라오게 설명(용어 풀어서).
- git add/commit/push 자동 처리. 위험한(되돌리기·삭제) 작업만 먼저 확인.
- 디제틱 위장(UI·메시지·코드 주석까지 포드라 안의 단어). legacy는 읽기 전용 참고.
- 배포는 main 머지. 코드 변경 후 lint+build 통과 확인.
