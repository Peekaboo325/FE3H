# 소지품 에셋 제작 명세 (인물 명부 · 소지품 탭)

빌더가 그림을 만들 때 보는 단일 기준 문서. 코드/명단 등재는 Claude Code 몫.

## 동작 방식 (요약)
- AI(Gemini)는 물건을 지을 때 **보유 그림 명단에서 제일 비슷한 것 하나**를 골라 딱지를 붙인다 (legacy emoji 선택과 같은 메커니즘).
- 고른 그림은 물건에 박제 — 새로고침해도 안 바뀜.
- 명단에 없는 답 → **공용 문양(폴백)** 으로 자동 처리. 화면은 절대 안 깨진다.
- **그림이 늘수록 선택이 정확해진다.** 전부 다 그릴 필요 없음 — 폴더에 들어오는 대로 한 장씩 살아난다.
- 카테고리는 **제작 가이드**(어디부터 그릴까의 지도)일 뿐, 시스템은 평평한 그림 명단만 안다.

## 공통 사양
- **정사각 512×512, 배경 투명 WebP.** (흰 배경본만 있으면 Claude Code가 배경을 따는 것도 가능 — 투명본이 최선)
- 위치: `public/assets/illust/items/` (파일명은 아래 표 그대로)
- 스타일(확정): **수채+선화의 사실적 소품화** — 밀랍 봉인 편지·코르크 약병·단검 샘플의 화풍이 기준. 차분한 채도(원색 금지), 묶음마다 같은 결 유지.
- 새 그림을 폴더에 넣으면 Claude Code에게 알려줄 것 → 명단에 등재해야 AI 선택지에 들어간다.

## 폴백 (최우선 1장)
| 파일 | 그릴 것 |
|---|---|
| `fallback.webp` | 공용 문양 — 어느 물건에 걸려도 어색하지 않은 것(묶인 보자기·매듭 등). **그림이 없는 동안 모든 카드가 이걸 쓴다.** |

## 집계
- 전체 **47장 + 폴백 1장** (끝까지 다 그리면 — 의무 아님)
- **1차 권장 12장(★)**: 서신 · 서책 · 반지 · 양초 · 술병 · 천 조각 · 찻잎 · 꽃 · 단검 · 성표 · 열쇠 · 수상한 꾸러미
- 순서·속도는 자유. 묶음 한 장(예: 3×3)으로 뽑아 낱장으로 잘라도 됨.
