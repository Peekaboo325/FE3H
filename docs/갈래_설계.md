# 갈래(岐) — 구상 단계 설계 (2026-07-06 확정)

> 막힘의 순간, 앱 안에서 "다음에 무슨 일이 벌어질 수 있나"의 갈래를 받아 고르는 **구상 단계**.
> 파이프라인의 빈 첫 칸을 채운다: **갈래(무슨 일) → 연출(어떻게) → 본문(렌더) → 교정(다듬기)** — 뒤의 셋은 기존.

## 1. 왜 (문제 정의 — 2026-07-06 빌더 상담)

- 빌더가 본문을 앱 밖에서 쓰게 된 근본 원인 = **혼자 다음 이야기를 짜기 힘들어서**. 구상을 도우러 바깥 AI에 갔더니 걔들이 **본문을 통째로** 줘버림 → 인터랙션이 일어나기 전에 이야기가 완성 → 앱은 보관소로 밀림.
- 진단(GPT 논의에서 채택): **"지금 앱은 사용자가 이미 다음 사건을 정해 와야 움직인다."** 막히는 순간 앱 안에 갈 곳이 없다.
- 처방: 앱 안에 구상 단계를 넣되, **바깥 AI의 병(산문 통째)이 재발하지 않게** 출력을 씨앗으로 강제.

## 2. 확정 결정 (빌더, 2026-07-06)

| 항목 | 결정 |
|---|---|
| 이름 | **갈래** (운명이 갈라지는 길들 — '운명의 장'과 같은 결) |
| 진입점 | **빈 입력창 = 전개 버튼이 '갈래' 버튼으로 변신** (버튼 0개 추가 — 막힌 순간=빈 칸에 정확히 나타남. 글을 쓰면 도로 '전개') |
| 선택 후 | **고른 갈래가 입력창에 채워짐** — 작가가 다듬은 뒤 직접 전개(주도권 유지, 파이프라인 무변경) |
| 모델 | **Flash로 시작** (서신·임무로 품질 검증·비용 티끌). 밋밋하면 다이얼 배선(연출 모델 패턴) |
| 후보 꼴 | **3개**, 각각 = 사건 1~2문장(event) + 무엇이 흔들리나 1문장(stakes). **완성 산문·대사 금지**, 서로 다른 결 강제(규칙로, 예시 없이 §3) |
| 재료 | 큰 줄기+최근 요약+최근 원문(`prepareConversation`) + 인물 요지(이름·신원·생사만 — 약력 통째 X) + 기록 지침. **미결 실타래는 2단계** |
| 호출 | `/api/story` 서브모드 `{ideate:true, story_id}` (Vercel 12함수 한도 — enrich·polish와 같은 결). **온디맨드만**(자동 호출 0) |

## 3. 흐름

막힘(입력창 빈 채) → [갈래] 버튼 → Flash가 갈래 셋 → compose 위 시트에 카드 3장 → 탭 = 입력창에 채움(편집 가능) → 전개(기존 흐름: 딥시크면 연출 콘티 게이트 → 실행 → 본문). 시트엔 다시 뽑기(↺)·닫기(X).

## 4. 지침 원칙 (프롬프트 — §3 예시 금지)

- 소설을 쓰지 마라 — 작가가 고를 **씨앗**만. 장면 묘사·대사·완성 문장 금지, 결말 단정 금지(어떻게 끝날지는 작가 몫).
- 이미 있는 것에서만 자라게: 등장한 인물·긴장·미결에서만. 새 인물·새 설정 지어내기 금지.
- 세 갈래는 서로 다른 방향(같은 종류의 변주 셋 = 실패) — 흔드는 인물·판(관계/정세/내면)을 다르게.
- 출력 = JSON `{"forks":[{"event","stakes"}×3]}` (모티프 대장의 Flash JSON 패턴).

## 5. 파일 지도

- `lib/ideate.mjs` — runIdeate(재료 조립→Flash JSON→검증). §3: 세계관·nameDict 미주입(이야기 자체만).
- `api/story.mjs`·`server/index.mjs` — `{ideate:true}` 서브모드.
- `src/App.tsx` — 버튼 변신(빈 입력=갈래)·갈래요청·시트(카드 3장·↺·X). `src/strings.ts` UI.ideate. `styles.css` .galae-*.
